parseHTML strategy benchmark

Compares five HTML-parsing strategies in this browser. Click Run benchmark to start. See the <details> below for what is measured and why.

What this measures

Five implementations of parseHTML(html) -> Array<Node>:

  1. singletonDOMParser — module-level new DOMParser(), reused.
  2. perCallDOMParser — fresh DOMParser per call.
  3. singletonTemplateReplaceChildren — module-level <template>, set innerHTML, snapshot children, then tpl.content.replaceChildren().
  4. singletonTemplateInnerHTMLEmpty — same template, cleanup is tpl.innerHTML = "".
  5. perCallTemplate — fresh <template> per call.

Inputs rotate over a corpus of 32 distinct strings per size bucket so we measure parsing, not warm caches around a single repeated string. Equivalence of all five outputs is asserted before timing begins.
